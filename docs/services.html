Services

Per-service documentation for Ledewire. Each section covers a service’s purpose, API surface, configuration, and relevant operational notes.

This page is a starting point. As services grow, each one can get its own dedicated page. For now, add a section below for each service.

api-gateway

Handles all inbound requests. Responsible for authentication validation, request routing, and rate limiting.

  • Repo: ledewire/api-gateway
  • Port (local): 3000
  • Owner:

Configuration

# Key environment variables (see .env.example)
PORT=3000
JWT_SECRET=...
RATE_LIMIT_WINDOW_MS=60000

core-service

Hosts the primary business logic for Ledewire. Coordinates domain operations and orchestrates calls to other services.

  • Repo: ledewire/core-service
  • Port (local): 3001
  • Owner:

data-service

Manages all interactions with the primary database. Exposes a stable internal API that other services consume — no service writes to the database directly.

  • Repo: ledewire/data-service
  • Port (local): 3002
  • Owner:

worker

Processes background jobs enqueued by other services. Handles anything that doesn’t need to complete within a request cycle.

  • Repo: ledewire/worker
  • Owner:
