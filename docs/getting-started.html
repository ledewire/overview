Getting Started

Everything you need to get Ledewire running locally and make your first contribution. This guide will take you from zero to a working development environment.

Prerequisites

Before you begin, make sure you have the following installed:

  • Node.js v18 or later — nodejs.org
  • Git 2.x or later
  • Docker & Docker Compose (for local services)
  • Access to the Ledewire GitHub organization

New to the team? Reach out to your onboarding contact to get access to the GitHub org and any required environment credentials before proceeding.

Installation

Clone the repository and install dependencies:

# Clone the repository
git clone https://github.com/ledewire/[repo-name].git
cd [repo-name]

# Install dependencies
npm install

# Copy local environment config
cp .env.example .env

Open .env and fill in the required values. All variables are documented inline in the example file.

Starting local services

Ledewire uses Docker Compose to run backing services (database, cache, etc.):

# Start all required services
docker compose up -d

# Confirm everything is running
docker compose ps

Your First Run

With dependencies installed and services running, start the development server:

npm run dev

The application will be available at http://localhost:3000. Hot-reloading is enabled by default in development mode.

First-time setup: Some features require seeded data. If you see errors about missing records, run npm run db:seed and reload.

Running tests

# Run the full test suite
npm test

# Watch mode (re-runs on file change)
npm run test:watch

# Lint the codebase
npm run lint

Next Steps

Now that you’re up and running, here’s where to go next:

  • Read the architecture overview to understand how the system fits together.
  • Review the contributing guide before opening your first PR.
  • Browse the ADRs to understand why key decisions were made.
  • Check the roadmap to see where the project is heading.
